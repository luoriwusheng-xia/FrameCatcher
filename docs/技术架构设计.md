# FrameCatcher 技术架构设计

## 项目概述

FrameCatcher 是一款基于 Electron + Vue3 + TypeScript 的跨平台桌面录屏应用，支持窗口录制、区域录制、音频录制、视频播放等功能。

**技术栈**: Electron 39 + Vue 3.5 + Vite 7 + TypeScript 5.9 + TailwindCSS 4.x

---

## 二、技术方案取舍

### 2.1 录屏方案

#### 选型过程

考虑过三种录屏技术路线：

| 方案 | 原理 | 放弃原因 |
|------|------|----------|
| A. `navigator.mediaDevices.getDisplayMedia` | 浏览器原生 API | 功能与 desktopCapturer 重叠，Electron 环境下优先使用原生 API |
| B. FFmpeg 命令行录制 (`gdigrab`/`avfoundation`) | 直接调用系统录屏接口 | 实时控制困难（暂停/恢复/区域预览），交互体验差 |
| **C. Electron `desktopCapturer` + `MediaRecorder`** | Electron 屏幕捕获 + Web API 录制 | ✅ **选定** — 交互体验好，系统窗口选择器原生支持，暂停/恢复控制灵活 |

#### 最终方案

- **窗口录制**: `desktopCapturer.getSources({types: ['window']})` 获取窗口列表，用户选择后通过 `navigator.mediaDevices.getUserMedia({ chromeMediaSource: 'desktop' })` 捕获对应窗口流
- **区域录制**: 通过全屏遮罩层让用户框选区域，获取坐标后裁剪视频帧（或利用捕获约束的 `mandatory` 参数限定区域）
- **音频录制**: 仅获取麦克风 + 系统音频流，无视频轨道

#### 音频处理细节

```
屏幕视频流 + 系统音频  ← desktopCapturer
          ↓
麦克风音频流  ← getUserMedia({ audio: true })
          ↓
AudioContext.createMediaStreamDestination() 合并音频
          ↓
MediaRecorder 录制 (webm 容器)
```

**已知限制**: macOS 系统音频捕获在某些场景下需要虚拟音频驱动（如 BlackHole），代码中需检测平台并做降级提示。

### 2.2 FFmpeg 集成方案

#### 选型过程

| 方案 | 原理 | 放弃原因 |
|------|------|----------|
| A. `ffmpeg-static` npm 包 | 安装时自动下载对应平台二进制 | 包维护状态差，版本更新不及时 |
| B. `fluent-ffmpeg` + `ffmpeg-static` | Node.js FFmpeg 封装 + 自动二进制 | fluent-ffmpeg 已停止维护，API 陈旧 |
| **C. 本地 FFmpeg 二进制 + `child_process.spawn`** | 手动下载 BtbN 构建版本，直接命令行调用 | ✅ **选定** — 可控性强，无 npm 依赖风险，版本自主管理 |

#### 最终方案

- **来源**: [BtbN/FFmpeg-Builds/releases](https://github.com/BtbN/FFmpeg-Builds/releases)
- **版本**: `ffmpeg-master-latest-win64-gpl` (static 静态链接版)
- **存放位置**:
  ```
  resources/
  ├── ffmpeg/
  │   └── ffmpeg.exe          (Windows)
  ├── ffmpeg-mac/
  │   └── ffmpeg              (macOS, 后续补充)
  └── ffmpeg-linux/
      └── ffmpeg              (Linux, 后续补充)
  ```
- **调用方式**: `child_process.spawn(getFFmpegPath(), args)`，不依赖任何 FFmpeg Node.js 包装库
- **打包**: electron-builder `extraResources` 配置自动将 `resources/ffmpeg` 打包进安装程序

#### FFmpeg 职责

| 功能 | FFmpeg 命令示例 |
|------|----------------|
| 格式检测 | `ffmpeg -i input.mkv` |
| 格式转码 | `ffmpeg -i input.webm output.mp4` |
| 视频分割 | `ffmpeg -ss 00:01:00 -t 30 -i input.mp4 output.mp4` |
| 提取音频 | `ffmpeg -i input.mp4 -vn -acodec copy output.aac` |
| 提取字幕 | `ffmpeg -i input.mkv -map 0:s:0 output.srt` |
| 获取信息 | `ffprobe -v quiet -print_format json -show_format -show_streams input.mp4` |

**重要约定**: 录制与转码完全分离。录制阶段用 MediaRecorder 输出 webm，录制完成后用户选择目标格式，再调用 FFmpeg 转码。不存在边录制边转码的场景。

### 2.3 视频播放方案

#### 选型过程

| 方案 | 原理 | 放弃原因 |
|------|------|----------|
| A. 所有格式走 FFmpeg 转码 | 播放前统一转码为 mp4 | 大文件转码耗时长，用户体验差 |
| B. 使用 VLC 插件或 WebChimera | 第三方播放器嵌入 | WebChimera 已停止维护，VLC 插件不稳定 |
| **C. HTML5 `<video>` 原生播放 + FFmpeg 兜底转码** | 先尝试原生播放，失败再转码 | ✅ **选定** — 覆盖 90% 场景，性能最优 |

#### 最终方案

```
用户打开视频
    ↓
尝试用 <video> 直接播放
    ↓
成功？ → 直接播放 (mp4/webm/mov/ogg 等)
失败？ → FFmpeg 转码为临时 mp4
              ↓
         转码完成后自动播放
```

- **原生支持格式**: mp4(h264/aac), webm, mov(QuickTime), ogg
- **需转码格式**: mkv, avi, flv, wmv, ts, 以及非 h264 编码的 mp4

### 2.4 配置存储方案

#### 选型过程

| 方案 | 原理 | 放弃原因 |
|------|------|----------|
| A. 平铺大对象 + `electron-store` | 所有配置字段放在一个大 JSON 中 | 扩展性差，字段增多后难以维护 |
| **B. 分区域 Class 化 + `electron-store`** | 每个配置区域独立 Class，统一管理校验和默认值 | ✅ **选定** — 类型安全，扩展性好，职责清晰 |

#### 最终方案

```
AppSettings (根管理器)
├── general: GeneralConfig    — 常规设置
├── audio: AudioConfig        — 音频设置
└── watermark: WatermarkConfig — 水印设置
    // 未来可扩展: shortcut, record, etc.
```

每个 Config Class 包含:
- `getDefaults()`: 返回默认值
- `setXxx(value)`: 强类型 setter，带校验
- `setPartial(data)`: 批量更新
- `_validators`: 校验规则表
- `toJSON()`: 序列化
- `reset()`: 恢复默认值

根管理器 `AppSettings` 提供:
- `setSection(section, data)`: 按区域更新
- `onChange(listener)`: 变更订阅
- `toJSON() / fromJSON()`: 全量序列化反序列化

持久化通过 `electron-store` 实现，主进程监听变更事件实时写入磁盘。

## 三、模块设计

### 3.1 录屏模块

```
RecorderView (录屏主界面)
├── 录制模式选择: [窗口] [区域] [音频]
├── 设备选择区: 麦克风 ▼ 扬声器 ▼ 摄像头 ▼
├── 预览区 / 窗口列表 / 区域遮罩
└── [开始录制]

录制流程:
[开始录制] → 3-2-1 倒计时 → MediaRecorder.start()
          → 弹出 RecordControls (可拖动控制条)
          → [暂停] [继续] [停止]
          → [停止] → 格式选择弹窗 → FFmpeg 转码 → 保存
```

#### 区域选择实现

- 新建无边框 `BrowserWindow`，全屏覆盖桌面
- 黑色半透明背景，中间矩形区域透明
- 矩形四角 8px 拖拽手柄，实时更新宽高
- 底部悬浮控制栏: 宽输入框 × 高输入框、常用尺寸下拉 (1920×1080, 1280×720, 800×600, 400×400)、取消/开始按钮

#### 录制控制条

- 无边框窗口，支持鼠标拖动
- 功能: 暂停/继续、停止、录制时长显示
- 录制中始终置顶

### 3.2 录制列表模块

```
RecordsView (录制列表)
├── 视图切换: [列表视图 ▼] [方格视图 ▼]
├── 录制项:

- 支持窗体放大缩小自适应布局

```
PlayerView (视频播放器)

## 四、状态管理

### Pinia Store 设计

```typescript
// stores/settings.ts
export const useSettingsStore = defineStore('settings', () => {
  const settings = reactive(new AppSettings())
  const isLoading = ref(true)

  async function load() {
    const data = await window.api.config.getAll()
    settings.fromJSON(data)
    isLoading.value = false
  }

  async function saveSection(section: string, data: Record<string, unknown>) {
    settings.setSection(section as any, data)
    await window.api.config.set(section, data)
  }

  return { settings, isLoading, load, saveSection }
})
```

### IPC 通道设计

```typescript
// Preload 暴露的 API
interface WindowAPI {
  config: {
    get: (key: string) => Promise<unknown>
    set: (key: string, value: unknown) => Promise<void>
    getAll: () => Promise<Record<string, Record<string, unknown>>>
  }
  ffmpeg: {
    check: () => Promise<boolean>
    convert: (input: string, output: string) => Promise<void>
    probe: (input: string) => Promise<FFprobeResult>
  }
  recorder: {
    getSources: () => Promise<DesktopCapturerSource[]>
    openAreaSelector: () => Promise<{x: number, y: number, width: number, height: number} | null>
  }
}
```

## 七、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| macOS 系统音频捕获受限 | macOS 用户无法录制电脑声音 | 检测平台，受限时弹窗提示用户可能需要安装虚拟音频驱动 |
| FFmpeg 二进制体积大 | 安装包增大 30~50MB | 仅打包当前平台二进制，分平台构建 |
| MediaRecorder 浏览器兼容 | 某些编码器不支持 | 使用标准 webm/vp9 编码，录制后统一转码 |
| 区域录制坐标精度 | 多显示器/高分屏坐标偏移 | 使用 `window.devicePixelRatio` 校准坐标 |
